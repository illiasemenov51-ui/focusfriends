# FocusFriends — патч «Отряды + Уведомления + Здоровье»

Накладывается **поверх** двух прошлых патчей (друзья+календарь, Terraria UI).

## Применить

```bash
cd ~/путь/к/focusfriends
unzip -o ~/Downloads/focusfriends-groups-health-patch.zip -d .
docker compose down -v      # новая БД health_db и новый init-script — нужен чистый volume
docker compose up --build
```

`down -v` обязателен в этот раз: `init-scripts/init-multiple-dbs.sh` изменился
(добавлена `health_db`), а он выполняется только при первой инициализации
volume — на существующем volume новая база не создастся.

## Что добавлено

### 1. Отряды (фронт — бэкенд уже был)
Во фронте не было ни одной кнопки для уже готового `GroupController`.
Добавил `GET /api/groups/mine` (отдать список своих групп — этого эндпоинта
не хватало) и вкладку **«ОТРЯДЫ»**: создать группу, вступить по id, список
участников с суммой очков (сортировка — тот же принцип, что в «Друзьях»).

### 2. Уведомления (бэкенд + связка)
`notification-service` существовал, но никто никогда не вызывал `/send` —
таблица уведомлений была бы вечно пустой. Добавил в `social-service`:
- `AuthServiceClient` — достаёт имя пользователя из `auth-service`
  (`GET /api/users/{id}`, публичный эндпоинт, без токена)
- `NotificationServiceClient` — best-effort отправка (если сервис недоступен,
  основной сценарий — заявка в друзья — не должен падать)

Подключил в двух местах `FriendshipService`:
- `sendRequest` → адресату: «{Имя} хочет добавить вас в друзья»
- `accept` → инициатору: «{Имя} принял(а) вашу заявку в друзья»

`notification-service` наконец подключён в `docker-compose.yml` (порт 8085→8084,
без него он вообще не поднимался, как раньше `social-service`), плюс роут
в `gateway-service`.

**Фронт для колокольчика/списка уведомлений не делал в этот раз** — бэкенд
теперь реально их создаёт, но UI для чтения решил оставить на следующий заход,
чтобы не размазывать этот патч ещё сильнее. Эндпоинты уже готовы:
`GET /api/notifications`, `PATCH /api/notifications/{id}/read`.

### 3. Здоровье — новый сервис `health-service` (Java, порт 8085→**8085**, поправка: см. ниже)
Отдельный микросервис по тому же паттерну, что остальные (Spring Boot,
Postgres, свой `X-User-Id`-интерцептор, свой `SocialServiceClient` для
проверки дружбы):

```
POST /api/health/checkins           — отметиться (upsert по дате: сон, энергия,
                                       стресс, настроение 1–5, заметка)
GET  /api/health/checkins?from=&to= — своя история
GET  /api/health/summary            — сводка за последние 7 дней:
                                       средние + loadIndex (0–100) + заметки
GET  /api/health/friends/{id}/summary — сводка друга (только если вы друзья
                                       И он не выключил "делиться")
GET  /api/health/privacy            — своя настройка приватности
PUT  /api/health/privacy             — { "shareWithFriends": true|false }
```

**Про приватность**: ты выбрал «полные цифры как в XP» — так и сделал, сводка
друга отдаёт точные числа. Добавил только тумблер **«Показывать мою сводку
друзьям»** (по умолчанию включён) — простой предохранитель, чтобы можно было
передумать в один клик, а не потому что не доверяю решению.

**Заметки в сводке** — это не диагноз, а пересказ твоих же собственных
отметок другими словами («судя по твоим отметкам, сна было мало») — сервис
не пытается поставить диагноз (это не его дело и не моё), только отражает,
что ты сам отметил за неделю.

Фронт: вкладка **«САМОЧУВСТВИЕ»** — форма отметки (слоты 1–5 в стиле
инвентаря Террарии для энергии/стресса/настроения), карточка сводки с
полосой нагрузки, тумблер приватности, выбор друга для сравнения.

### 4. Полировка UI (по мелочи, попутно)
- Новый переиспользуемый `EmptyState` — пустые состояния теперь в виде
  «пустого сундука» вместо обычного текста; применил в «Задачах»,
  «Привычках», «Друзьях», «Отрядах».
- Мобильная адаптация вкладок дашборда уже была на месте
  (`variant="scrollable"`), в этот заход не трогал.

## Порт health-service

Обрати внимание: `notification-service` — 8084, `health-service` — 8085.
В `docker-compose.yml` и `gateway-service/application.yml` уже прописано
правильно, но если руками поднимаешь `health-service` локально (не через
compose) — не перепутай порт с notification-service.

## Известные ограничения на конец этого патча

- UI для чтения уведомлений (колокольчик) — не сделан, эндпоинты готовы
- Achievements/ачивки — ещё не начинал (следующий заход, как договаривались)
- `health-service` в этот раз не покрыт `docker compose down -v` автоматически
  на старом volume — см. инструкцию выше

Собрано и протипчекано (`tsc -b`, `oxlint`, `vite build`) для фронта — ошибок
нет. Java вручную проверен на баланс скобок и соответствие DTO ↔ фронт-типов
(нет доступа к Maven Central в этой среде, чтобы прогнать полную сборку) —
собери `docker compose up --build` и покажи лог, если что-то не взлетит.
