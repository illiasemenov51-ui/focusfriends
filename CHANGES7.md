# FocusFriends — патч «Большие кнопки + быстрый доступ к осанке + звук»

Только фронтенд, накладывается **поверх** всех прошлых патчей.

## Применить

```bash
cd ~/путь/к/focusfriends
unzip -o ~/Downloads/focusfriends-quick-posture-patch.zip -d .
docker compose up --build frontend
```
(бэкенд не менялся)

## Что сделал

### 1. Большие кнопки
`MuiButton`/`MuiIconButton` в теме теперь крупнее по умолчанию (`size="large"`,
больше padding, чуть крупнее шрифт) — по всему приложению разом, без правки
каждого экрана отдельно.

### 2. Круглая кнопка «проверить осанку» на главном экране
Наверху дашборда (видна на любой вкладке) — кружок с иконкой позвоночника.
В обычном состоянии позвонки «сутулые» (сдвинуты, персиковый цвет). Нажал —
они плавно выпрямляются в вертикальную линию (зелёный), и через мгновение
приложение само:
1. переключается на вкладку «САМОЧУВСТВИЕ»
2. внутри неё — на под-вкладку «ОСАНКА»
3. включает камеру и запускает распознавание

Всё это — через общий React-контекст (`QuickActionsContext`), а не проп-дриллинг
через три уровня компонентов.

### 3. Настройка звука предупреждения
Раньше был зашит один звук. Теперь на вкладке «Осанка» — отдельная карточка:
- 3 встроенных пресета (короткий сигнал / два коротких / колокольчик),
  синтезируются на лету через Web Audio, без файлов
- **Загрузить свой звук** — обычный `<input type=file accept=audio/*>`,
  файл превращается в base64 и сохраняется в `localStorage` браузера
  (не на сервере — это чисто локальная настройка устройства)
- Кнопка «Прослушать» — проверить сигнал, не дожидаясь плохой осанки

### 4. «Калории уже посчитаны»
На вкладке «Калории» — если за сегодня уже есть отметка калорий, наверху
зелёный баннер: «Калории на сегодня уже посчитаны: N ккал» вместо того,
чтобы молча показывать пустую форму.

## Что осталось на следующий заход (по твоей же просьбе разбить)

- **Онбординг при регистрации** (когда обед, когда сон, режим дня) —
  отдельная фича, нужно менять флоу регистрации + добавлять поля на бэкенде
- **Мультиязычность** — нужна библиотека i18n + вынести все строки интерфейса
  в словари; при таком объёме экранов это отдельный большой заход, не хочу
  делать наспех и потом чинить кривые переводы
