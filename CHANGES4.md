# FocusFriends — патч «API-токены для Shortcuts (Apple Health)»

Накладывается **поверх** всех прошлых патчей.

## Применить

```bash
cd ~/путь/к/focusfriends
unzip -o ~/Downloads/focusfriends-api-tokens-patch.zip -d .
docker compose down -v      # новая таблица api_tokens в auth_db
docker compose up --build
```

## Почему не напрямую из HealthKit

HealthKit — API уровня iOS, доступен только нативным приложениям (Swift),
у браузера/веб-фронта к нему в принципе нет доступа — так устроена iOS,
это не ограничение нашего стека. Писать нативное приложение на Swift я не
могу — в этой среде нет Xcode/iOS-симулятора, чтобы его собрать и
протестировать.

Рабочий путь без нового приложения — **iOS Shortcuts**: сам умеет читать
Health и делать HTTP-запросы. Не хватало только способа его авторизовать —
обычный access-токен живёт 15 минут, а автоматизация должна работать
самостоятельно днями/неделями.

## Что добавлено

### Персональные API-токены (`auth-service`)
Как GitHub Personal Access Token: создаёшь в приложении, копируешь один раз
(хранится только SHA-256-хэш, не сам токен), используешь как обычный Bearer.
Не истекает сам — только по отзыву.

```
POST   /api/auth/api-tokens         { "name": "iPhone Shortcuts" } → токен виден один раз
GET    /api/auth/api-tokens         — список своих токенов (без секрета)
DELETE /api/auth/api-tokens/{id}    — отозвать
```

### Gateway понимает токен двух видов
`JwtAuthenticationGlobalFilter` теперь смотрит на префикс:
- обычный JWT (веб-сессия) — проверяется локально по подписи, как раньше
- `ff_pat_...` (персональный токен) — гейтвей дёргает
  `GET /api/auth/api-tokens/validate` в auth-service (внутренний вызов,
  заголовок `X-Api-Token`, минуя обычный роутинг) и получает `userId`

Дальше для всех остальных сервисов (`health-service` и т.д.) разницы нет —
они как получали `X-User-Id` от gateway, так и получают, независимо от
способа авторизации.

### Фронт: вкладка «САМОЧУВСТВИЕ» → блок «интеграции»
- Создать токен с именем, скопировать (показывается один раз)
- Список активных токенов с датой последнего использования, кнопка отозвать
- Разворачиваемая инструкция **прямо в приложении**, как собрать сам Shortcut

## Как собрать сам Shortcut (то же, что в инструкции в приложении)

1. Приложение **Быстрые команды** → создать новую
2. «Получить образцы состояния здоровья» → тип **Сон**, диапазон — прошлая ночь
3. Посчитать часы сна из образца
4. (по желанию) «Запросить ввод» — число 1–5 для энергии/стресса/настроения,
   HealthKit не хранит субъективные показатели, кроме сна
5. «Получить содержимое URL»:
   - URL: `http://<твой-gateway-host>:8080/api/health/checkins`
   - Метод: `POST`
   - Заголовок: `Authorization: Bearer <токен из приложения>`
   - Тело (JSON): `sleepHours`, `energyLevel`, `stressLevel`, `moodLevel`
6. Автоматизация → «Время суток» (например, каждое утро в 9:00) → запускать
   этот Shortcut

## Известные ограничения

- HealthKit хранит только объективные метрики (сон, шаги, пульс и т.д.) —
  «энергия/стресс/настроение» либо вводятся вручную в Shortcut (быстрый
  диалог с числом), либо можно расширить на сон + HRV из Apple Watch как
  косвенный индикатор стресса — если понадобится, это отдельный заход.
- Токен виден только в момент создания — потерял, значит отозвал и создал
  новый (стандартная практика для PAT, не баг).
- Гейтвей теперь делает сетевой вызов в auth-service для каждого запроса
  с `ff_pat_`-токеном (JWT остаётся быстрым локальным путём) — для личного
  проекта не критично, при реальной нагрузке стоило бы кэшировать
  результат на несколько минут.
