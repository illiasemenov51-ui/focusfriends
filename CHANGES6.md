# FocusFriends — патч «Упорядочить + Калории + Осанка»

Накладывается **поверх** всех прошлых патчей.

## Применить

```bash
cd ~/путь/к/focusfriends

# сначала удали переименованные/устаревшие файлы (иначе останутся дублирующие
# сущность старые файлы рядом с новыми — Spring их даже не подхватит, но зачем мусор)
while read -r f; do rm -f "$f"; done < <(unzip -p ~/Downloads/focusfriends-health-v2-patch.zip DELETE_THESE_FILES.txt)

unzip -o ~/Downloads/focusfriends-health-v2-patch.zip -d .
rm DELETE_THESE_FILES.txt   # больше не нужен, попал в корень при распаковке

docker compose down -v      # переименована таблица health_privacy_settings → health_settings
docker compose up --build
```

## Упорядочил

Раздел «Самочувствие» разросся (чек-ин + приватность + токены Shortcuts всё
в одном файле) — разбил на подпапку `frontend/src/components/health/` с
явным разделением по смыслу:

- `HealthCheckinTab.tsx` — ежедневная отметка + твоя недельная сводка + сравнение с другом
- `CaloriesTab.tsx` — цель по калориям + прогресс за неделю
- `PostureSection.tsx` — контроль осанки через камеру
- `HealthSettingsTab.tsx` — приватность + интеграции (Shortcuts)
- `shared.tsx` — общие кусочки (карточка сводки, выбор 1–5, цвета) — раньше
  дублировались

Сама вкладка «САМОЧУВСТВИЕ» теперь показывает свои под-вкладки вместо одной
длинной прокручиваемой страницы.

Также на бэкенде объединил `HealthPrivacySettings` → **`HealthSettings`**
(таблица `health_settings`) — она и раньше хранила по сути "личные настройки
раздела", просто называлась только про приватность. Теперь там же живёт
`calorieGoal`. Один DTO `GET/PUT /api/health/settings` вместо
`/api/health/privacy`.

## Калории

- В форме чек-ина — необязательное поле «калорий за день»
- Отдельная вкладка **«КАЛОРИИ»**: задать дневную цель, увидеть средние
  калории за 7 дней и прогресс относительно цели
- В недельной сводке (в том числе у друга, если он делится) появилась
  строка `🍽️ N ккал`, и note-наблюдение, если расхождение с целью ≥15%

Специально не стал делать полноценный дневник приёмов пищи (завтрак/обед/
ужин по отдельности, поиск продуктов и т.п.) — это отдельный, гораздо более
объёмный модуль. Если нужно — дай знать, сделаю следующим заходом.

## Осанка

Новая вкладка **«ОСАНКА»** внутри «Самочувствия»:

- Кнопка «Включить контроль осанки» — запрашивает доступ к камере
- Кнопка «Откалибровать» — нажимаешь, сидя ровно, дальше система сравнивает
  текущее положение с этим эталоном
- Как только просидел скособоченным дольше ~3 секунд подряд — короткий звуковой
  сигнал (раз в 8 секунд, чтобы не задолбать)
- Тумблер звука — можно оставить только визуальный индикатор
- Кнопка «Выключить камеру» — камера физически гаснет (поток останавливается,
  не просто скрывается превью)

**Технически**: распознавание позы — `@mediapipe/tasks-vision`
(Google MediaPipe Pose, облегчённая модель), работает полностью в браузере
через WASM. Видео **никогда никуда не отправляется** — ни на бэкенд, ни на
сторонние серверы, ни в логи. Это принципиально: единственный вариант, при
котором такая фича вообще допустима — данные не покидают устройство. Библиотека
подключена через `import()`, поэтому её ~135 КБ не загружаются, пока не
откроешь вкладку «Осанка».

Эвристика намеренно простая (v1): соотношение позиции носа к средней линии
плеч, нормированное на ширину плеч (устойчиво к тому, что ты то ближе, то
дальше от камеры). Не идеально для всех типов осанки (например, наклон
вбок или сутулость без опускания головы поймает не так хорошо) — если
станет актуально, можно усложнить эвристику или добавить больше точек.

## Известные ограничения

- Не смог протестировать сам детектор позы вживую — в этой среде нет камеры
  и браузера. Собрано и типизировано без ошибок, но реальную точность
  калибровки/порогов стоит подстроить под себя эмпирически (константы
  `BAD_POSTURE_THRESHOLD` и `BAD_POSTURE_SUSTAIN_MS` в начале
  `PostureSection.tsx` — если сигналит слишком часто/редко, подкрути их)
- Модель MediaPipe грузится с CDN (`storage.googleapis.com`,
  `cdn.jsdelivr.net`) — нужен доступ в интернет с машины, где открыт браузер
  (не influences сам docker compose, просто браузеру нужен внешний доступ)

Собрано и протипчекано (`tsc -b`, `oxlint`, `vite build`) — ошибок нет.
Java проверен вручную на баланс скобок и соответствие DTO ↔ фронт-типов.
